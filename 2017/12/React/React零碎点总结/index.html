21.immutable：数据创建后不会被改变，可用于react和redux的性能优化

优点：

  • 减少内存使用
  • 并发安全
  • 降低项目复杂度
  • 便于比较复杂数据，定制shouldComponentUpdate方便
  • 时间旅行功能方便
  • 函数式编程

缺点：

  • 学习成本
  • 库的大小
  • 对原有项目入侵严重，所以新项目使用比较合适
